# stock-report — レポート形式と分析原則

## レポートテンプレート（厳守）

```
## 株式分析 YYYY-MM-DD

### 総括
- [市場全体の状況 1行: VIX値・S&P500・USDJPY・米10年債]
- [ポートフォリオコンテキスト 1行: 評価額・現金・信用倍率]
- [相関リスク評価 1行: avg相関値・最大ペア]

## ポートフォリオ分析

### 相関行列
[8×8行列テーブル]
- 平均相関の解釈（1文）
- 分散効果が高い/低い銘柄の指摘（1文）
- 最大相関ペアの指摘と解釈（1文）

### ストレステスト
[4シナリオのテーブル: GFC/COVID/Aug/JPY]
- 最大損失シナリオの解釈（1文）

---

## 銘柄別詳細

### TICKER 銘柄名 — アクション（含み損益率）

**同一ティッカー複数ポジションがある場合**: `### TICKER 銘柄名 — Nポジション` でグループ化し、`#### TICKER 銘柄名（現物/信用/現物①） — アクション（PnL%）` で列挙する。ポジションタイプ（現物/信用）をタイトルに必ず含める。全ポジション数が portfolio.yaml の `holdings` 数と一致すること。

| 項目 | 値 |
|------|-----|
| 現在値 | ¥X,XXX（取得 ¥X,XXX × X株） |
| 含み損益 | ±XX.X%（±¥X,XXX） |
| 目標 | ¥X,XXX（アナリストX名 buy/hold/sell / 現在よりXX%上/下） |
| シグナル | rule strength BUY/SELL（複数時は列挙） |
| トレンド | trend_state / RSI XX.X / 特記事項 |
| VaR(1日最大損失目安) | 95%の確率で1日の下落は **-X.XX%以内** |
| Kurtosis | X.XX（3以上でfat tail注意） — バックテストJSONから必ず取得 |
| Sharpe(戦略の収益性) | X.XX（1.0以上で良好） |
| Sharpe CI(戦略の信頼性) | [X.XX, X.XX] — 0を含むか否か |
| WF判定(戦略の安定性) | robust / unstable(n/N) / insufficient_data(n/N) |
| 相関 | 最大ペア / β=X.XX（portfolio_analytics.jsonから取得） |

**分析**: 以下の全要素を必ず含めること（1要素1文、合計5-7文）:
1. **価格と損益**: 現在値の水準、含み損益の評価
2. **シグナルとトレンド**: アクティブなシグナルが示す方向性、トレンド状態との整合性
3. **リスク**: VaRの大きさの意味（他銘柄との比較）、Kurtosisが高い場合はfat tailリスクの指摘
4. **バックテスト**: Sharpeの良否、CIが0を含む場合は「統計的に有意な収益力は確認できない」と明記
5. **WF判定**: 判定が unstable/insufficient_data の場合は「過去データでの戦略の安定性は低い」と明記
6. **相関**: 他銘柄との連動性、分散効果への貢献度
7. **マクロ**: 為替・金利の影響（輸出株なら円安追い風、金ETFなら米金利逆風など）

**判断**: 以下のリスト形式で必ず記述すること:
- アクション: **保有継続 / 追加買い / 一部売却 / 全株売却 / 損切り準備**
- 数量: **X株**（必ず100株単位）
- 損切りライン: **¥X,XXX**（BB下限 または 取得単価比-X%）
- 理由: 1文で判断根拠を記述
```

## 分析完了チェックリスト（レポート提出前に自己チェック必須）

各銘柄の分析に以下の全要素が含まれているか確認すること:

- [ ] データテーブルに **10行すべて** が存在する（現在値〜相関）
- [ ] データテーブルの数値はバックテストJSON/portfolio_analytics.jsonから**実際に取得**した値である
- [ ] 分析に **7要素すべて**（価格損益/シグナルトレンド/リスク/バックテスト/WF/相関/マクロ）が含まれている
- [ ] 判断に **アクション種別・数量・損切りライン・理由** の4要素が含まれている
- [ ] 数量は必ず **100株単位**
- [ ] 「データ不足」「確認できず」などのあいまい表現ではなく、具体的な数値と判定を使っている

## 時価の取得（必須）

- **signal_engine.json の `indicators.close`（yfinance）を時価の正とする**
- portfolio.yaml の `current_price` は参照しない
- yfinance が未取得の場合は取得単価で代替し、その旨を注記

## 分析原則

- **断定的に書く**: 数量を明示（「100株単位で追加買い」）
- **単元株を厳守**: 数量は必ず100株単位。ETF等は保有数量確認
- **ラギング指標より直近価格を優先**
- **信用期限は絶対条件**: 残日数 < 60日の信用ポジションは必ず言及
- **数値で語る**: 「高い/低い」でなく具体的な数値で
- **見送りも明示**: 動かない銘柄も「見送り」と書く

## マクロ環境の読み方

| 指標 | 閾値 | 日本株への影響 |
|------|------|--------------|
| VIX | >= 25 | リスクオフ。BUY保留、HOLD/SELL優先 |
| VIX | <= 13 | 過信（楽観過多）。高値掴み注意 |
| S&P500前日 | <= -1.5% | 日本株寄付き売り圧力。SELL/HOLD |
| S&P500前日 | >= +1.0% | 寄付き買い追い風。BUY後押し |
| USD/JPY | >= 155円 | 円安。輸出株（任天堂・IHI等）に追い風 |
| USD/JPY | <= 140円 | 円高。輸出株逆風、内需株有利 |
| 米10年債 | >= 4.5% | 高金利継続。成長株バリュエーション圧迫 |

## アナリスト目標株価の読み方

| 条件 | 解釈 |
|------|------|
| 現在値がアナリスト目標の **75%以下**（乖離+25%超） | 割安シグナル。BUY/HOLD後押し |
| 現在値がアナリスト目標の **115%以上**（乖離-15%超） | 割高シグナル。SELL/利確検討 |
| アナリスト「買い/強気買い」+ テクニカル強気 | コンビクション高い BUY |
| アナリスト「売り」+ RSI > 70 | 複合SELL |
| 目標株価レンジ 高値÷低値 > 3x | アナリスト見解分かれる。過信禁止 |
| アナリスト人数 < 3人 | カバレッジ薄。目標株価の信頼性低 |
